## 📊 关键数据流示例

### 示例：控制回转电机移动到 90°

```
用户输入：90°
   ↓ ROS 2 Topic
应用层换算：250 units
   ↓ CiA402 状态机
CANopen SDO：写入 0x607A = 250
   ↓ Linux write()
CANopen PDO：发送控制字 0x0F + 位置 250
   ↓ SYNC 触发
CAN 帧：0x203 [0x0F, 0x00, 0xFA, 0x00, 0x00, 0x00]
   ↓ USB-CAN
差分信号：CAN_H/CAN_L 2V
   ↓ CAN 总线
电机驱动器：接收 → 解码 → 执行
   ↓ 物理运动
输出轴旋转：250 units (90°)
```

## 🚀 快速开始指南

### 1. 发送位置指令

```bash
# 控制回转电机移动到 90 度
ros2 topic pub /slewing/target_position std_msgs/msg/Float32 "data: 90.0" --once
```

### 2. 监控电机状态

```bash
# 查看当前位置
ros2 topic echo /slewing/crane_position

# 查看电机状态
ros2 topic echo /slewing/crane_status
```

### 3. 查看 CAN 总线流量

```bash
# 监控所有 CAN 帧
candump can0

# 过滤特定节点（回转电机 Node ID 3）
candump can0,203:7FF  # RxPDO1
candump can0,183:7FF  # TxPDO1
candump can0,603:7FF  # SDO 请求
candump can0,583:7FF  # SDO 响应
```

## 🛠️ 故障排查快速参考

### 问题1：电机不响应位置指令

**排查步骤**：
1. 检查 ROS 2 Topic 是否发布成功：`ros2 topic echo /slewing/target_position`
2. 查看电机状态字：`ros2 topic echo /slewing/crane_status`
3. 检查 CAN 总线：`candump can0`
4. 确认电机状态机在 "Operation Enabled" (0x0027)

**相关文档**：主文档第3章（CiA402 状态机）

---

### 问题2：角度与实际运动不匹配

**排查步骤**：
1. 确认减速比配置正确：回转/小车=10.0，吊钩=20.0
2. 检查转换函数：`angle_to_position()` 的计算逻辑
3. 验证 `target_units_per_rev` 参数（默认 10000）

**相关文档**：主文档第3章 + 流程图（数据格式转换）

---

### 问题3：多电机不同步

**排查步骤**：
1. 确认使用了 SYNC 同步帧（0x080）
2. 检查是否使用 `sync_trajectory_action_server`
3. 验证 PDO 传输类型配置为同步模式（0xFF）

**相关文档**：流程图（多电机同步示例）
