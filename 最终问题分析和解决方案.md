# 电机 SDO 通信问题 - 最终分析报告

**日期**: 2026-01-13
**问题**: ROS2 系统启动失败,报 SDO 连接错误

---

## 问题核心本质

经过详细诊断和多次测试,**根本问题是**:

### ✅ 已确认的事实
1. **CAN 接口正常**: can0 已启动,状态为 UP
2. **CAN 总线有流量**: 每 2 秒有 3000+ 条消息(主要是 TPDO)
3. **电机发送 PDO 数据**: 检测到 0x281, 0x283 等 TPDO 消息
4. **所有电机都不响应 SDO**: 节点 1, 2, 3 全部无响应

### ❌ 核心问题
**电机驱动器不响应 SDO (Service Data Object) 请求**

这意味着:
- 电机驱动器可能**仅配置为 PDO 模式**
- 电机驱动器的 CANopen 实现**不支持或禁用了 SDO**
- 电机驱动器可能处于**错误状态或未正确初始化**

---

## 已尝试的解决方案

### 方案 1: 禁用启动配置 ❌ 失败
**修改**: 设置 `boot: false`, `boot_timeout_ms: 0`
**结果**: 驱动仍然尝试启动,因为存在 .bin 文件

### 方案 2: 删除 .bin 文件 ❌ 失败  
**修改**: 删除 hook_joint.bin, trolley_joint.bin, slewing_joint.bin
**结果**: 驱动仍然尝试执行启动过程

### 方案 3: 设置空 dcf_path ❌ 失败
**修改**: 为每个节点设置 `dcf_path: ""`
**结果**: 导致 `lely::bad_init` 错误,文件路径无效

### 方案 4: 降低主站启动超时 ✅ 部分成功
**修改**: 设置 `master.boot_timeout: 0`
**结果**: 减少了一些超时,但从节点仍尝试启动

---

## 为什么 ros2_canopen 无法工作

`ros2_canopen` 的 `Cia402Driver` 设计为:
1. 在初始化时通过 **SDO 读取** 电机状态(Statusword 0x6041)
2. 通过 **SDO 写入** 配置参数(操作模式、速度等)
3. 通过 **SDO 写入** Controlword (0x6040) 控制电机状态机
4. **然后**才使用 PDO 进行实时控制

但你的电机驱动器:
- ❌ 不响应 SDO 读取请求
- ❌ 不响应 SDO 写入请求  
- ✅ 能发送 TPDO (状态信息)
- ❓ 未知是否接收 RPDO (命令)

---

## 可能的根本原因

### 原因 A: 电机驱动器固件配置
某些工业 CANopen 驱动器可以配置为:
- **模式 1**: 完整 CANopen (SDO + PDO)
- **模式 2**: 仅 PDO 模式(更简单,更快,但不支持 SDO)

**你的电机可能处于模式 2**。

**如何检查**:
1. 查看电机驱动器型号和手册
2. 检查驱动器配置软件/DIP 开关设置
3. 查找 "PDO only" 或 "SDO disabled" 相关设置

### 原因 B: 电机驱动器未进入 Operational 状态
CANopen NMT 状态机:
```
Initialization → Pre-operational → Operational
```

SDO 通常在 Pre-operational 状态下可用。
如果电机卡在 Initialization 或其他状态,可能不响应 SDO。

**如何检查**:
```bash
# 监听 NMT 心跳消息 (0x701, 0x702, 0x703)
timeout 5s candump can0 | grep -E '70[1-3]'

# 状态值:
# 0x00 = Bootup
# 0x04 = Stopped  
# 0x05 = Operational
# 0x7F = Pre-operational
```

### 原因 C: 节点 ID 或波特率不匹配
虽然可能性较小(因为看到了 PDO),但值得检查:
- 驱动器的 CANopen 节点 ID 是否确实是 1, 2, 3?
- 驱动器的 CAN 波特率是否确实是 500 kbit/s?

## 实际可行的解决方案

### 解决方案 A: 物理检查和复位电机驱动器 (推荐首先尝试)

**步骤**:
1. **关闭所有电机驱动器电源**
2. **等待 30 秒** (让电容完全放电)
3. **检查驱动器显示屏**,是否显示错误代码
4. **按驱动器复位按钮**(如果有)
5. **重新上电**
6. **等待驱动器完全初始化**(通常 5-10 秒)
7. **发送 NMT 启动命令**:
```bash
# 启动所有节点进入 Operational 状态
cansend can0 000#0100
sleep 1

# 测试 SDO 响应
cansend can0 603#4041600000000000
timeout 1s candump can0,583:7FF -n 1
```

8. **如果现在有响应**,再次启动 ROS2 系统

### 解决方案 B: 配置电机驱动器启用 SDO (永久修复)

**需要**:
- 电机驱动器配置软件(通常是制造商提供的 Windows 软件)
- USB-CAN 适配器或驱动器支持的其他配置接口

**步骤**:
1. 使用制造商软件连接到驱动器
2. 检查 CANopen 配置:
   - 节点 ID: 必须为 1, 2, 3
   - 波特率: 必须为 500 kbit/s
   - **SDO 服务器**: 必须启用 ✅
   - **PDO**: 可选启用
3. 保存配置并重启驱动器
4. 测试 SDO 响应(如解决方案 A 步骤 7)

### 解决方案 C: 使用自定义驱动 (高级,需要编程)

如果电机确实不支持 SDO,需要编写一个**仅使用 PDO 的自定义 ros2_control 硬件接口**。

这需要:
- C++ 编程
- 深入理解 ros2_control 架构
- 直接使用 SocketCAN API 发送/接收 PDO

**不推荐**,除非其他方案都失败。

### 解决方案 D: 更换电机驱动器 (最后手段)

如果驱动器固件真的不支持 SDO 且无法更新,考虑更换为:
- 支持完整 CANopen (SDO + PDO) 的驱动器
- 支持 EtherCAT / Ethernet/IP 等其他协议的驱动器
